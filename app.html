Field Data Enabler PN App for Linux


 

  • Enabling digital use cases (e.g. Overall Equipment Efficiency)

  • Field data access w/o PLC programming (due to generic data provisioning in an OPC UA server)

  • No additional hardware or upgrade needed (due to use of PROFINET and IO-Link standard)

  • Field data access for IT and Edge application

  • No performance impact to the PLC due to communication bypass (Y) solution


Why FIELD DATA ENABLER PN App is leveraging OPC UA?

  • OPC UA is capable to provide standardized Information models in a modular way

  • The IO-Link device identifiers found during scan are combined with the device description files to create a device specific information model

  • This is done without reading all device data (as less load impact to the network as possible)

  • The OPC UA client can efficiently decide during runtime which data is used (all data is available in the server but the client chooses a subset)
